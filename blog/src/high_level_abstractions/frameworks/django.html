Stateful vs Stateless

Stateful

An Stateful process allow clients/users to store, record, and return to
already established information and processes over the internet

So, In stateful process, what the client/user can do with it?

But, whats the process actually storing or recording? **(there might be actions, preferences, etc...)

In a stateful process, what whill happend if the user/client "lost connection"

First, as we can see in the previous image, all user/client information is stored in the process
So, what will happen when the client/user reconnects to that process?

So in that case, whenever the user tries to reconnect to the process and try to get information
about where he left off, the process will retrieve that information from its database, or txt file
or whatever it is using for storing data, then it'll send it back to the client, so the client
will be where he left off

Stateless

A stateless process does not retain information about the user's previous interactions.
